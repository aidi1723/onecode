## Goal

Upgrade OneCode from a timeout-aware kernel skeleton into a physically governable action system. Every executable operation must be represented as an auditable `ActionIntent`, pass through `LogosGate.preflight()`, and produce structured evidence before any side effect is allowed.

## Scope

v0.2-beta implements the smallest real sovereignty surface:

- auditable action intents
- a permission matrix keyed by current `HexagramStatusCode`
- a preflight gate returning `allowed`, `denied`, or `halted`
- one real side-effect: `PathGuard.write_text()`
- append-only manifest history

v0.2-beta does not include a TUI, model adapter, benchmark harness, strategic skill layer, shell execution, pytest execution, delete operation, directory traversal API, or general virtual filesystem.

## Architecture

New and modified modules:

```text
src/onecode/kernel/
  action_intent.py        # ActionIntent, ActionType, payload validation
  permission_matrix.py   # state + action policy decisions
  path_guard.py          # scoped write_text physical side effect
  logos_gate.py          # preflight plus existing timeout boundary
  checkpoint.py          # append-only manifest history
  runner.py              # intent-driven execution
```

### ActionIntent

`ActionIntent` is the only accepted representation of requested work.

Required fields:

- `action_type`
- `payload`

Initial action types:

| Action Type | Physical execution in v0.2 | Purpose |
| --- | --- | --- |
| `noop` | yes, no side effect | smoke compatibility |
| `write_text` | yes | only real write side effect |
| `execute_pytest` | no | permission matrix test target |
| `bash_execution` | no | denied high-risk test target |

`write_text` payload:

```json
{
  "path": "src/generated_asset.py",
  "content": "..."
}
```

Invalid action type or missing payload fields must be rejected before execution.

### PermissionMatrix

The permission matrix maps:

```text
HexagramStatusCode + ActionType -> PermissionDecision
```

Initial policy:

| State | Action | Decision |
| --- | --- | --- |
| `111111` | `noop` | `allowed` |
| `111111` | `write_text` | `allowed` |
| `111111` | `execute_pytest` | `denied` |
| `111111` | `bash_execution` | `denied` |
| any state | unknown action | `halted` |

The matrix may define richer state behavior later. v0.2 only needs enough policy to prove that actions are no longer raw callables and that high-risk tool categories are blocked before execution.

### LogosGate Preflight

`LogosGate.preflight(context, intent)` returns:

```json
{
  "decision": "allowed",
  "reason": null,
  "intent_type": "write_text",
  "state": "111111",
  "evidence_required": ["path", "sha256"]
}
```

Decision meanings:

- `allowed`: execution may continue through the approved executor.
- `denied`: policy denied the action but the run can continue or report a non-partial denial.
- `halted`: sovereignty breach or invalid intent; the run must stop and write evidence.

Standard reasons:

| Reason | Meaning |
| --- | --- |
| `permission_denied` | action is known but disallowed in the current state |
| `sovereignty_breach` | path or intent violates hard workspace sovereignty |
| `invalid_intent` | intent cannot be validated |
| `http_timeout` | bounded execution exceeded timeout |

### PathGuard

`PathGuard.write_text(workspace_root, relative_path, content)` is the only physical write API in v0.2.

Rules:

- `relative_path` must be relative.
- resolved target path must stay inside `workspace_root`.
- parent directories for allowed targets may be created.
- writes are atomic: temporary file, flush, fsync, rename.
- successful write returns SHA256 of the final file.
- rejected paths perform no target write.

Allowed examples:

```text
src/generated_asset.py
docs/output.md
```

Rejected examples:

```text
../outside.txt
/absolute/path.txt
.git/config
.env
.env.local
.gitignore
pyproject.toml
```

Root configuration writes are denied in v0.2. The denied root config set is:

- `.env`
- `.env.*`
- `.gitignore`
- `pyproject.toml`
- paths under `.git/`

### Append-Only Manifest

v0.1 rewrites `manifest.json` with one checkpoint record. v0.2 still writes `manifest.json` atomically as a full JSON document, but its logical checkpoint history is append-only.

When `write_checkpoint()` is called:

1. Read existing manifest if it exists.
2. Preserve existing checkpoint records.
3. Append the new checkpoint record.
4. Atomically rewrite the manifest.

Each checkpoint record includes:

- path
- sha256
- turn index
- status
- partial flag
- reason
- intent type when available
- decision when available

### Runner

The runner must stop creating raw anonymous actions as the primary execution unit. It should create `ActionIntent` values and ask `LogosGate.preflight()` before side effects.

v0.2 runner behavior:

- default CLI smoke uses `noop`
- CLI may accept `--write-path` and `--write-content` for `write_text`
- denied or halted preflight writes checkpoint and ledger evidence
- allowed `write_text` invokes `PathGuard.write_text()` and records returned SHA256
- timeout behavior from v0.1 remains intact

## Data Flow

```text
task input
  -> ActionIntent
  -> LogosGate.preflight(context, intent)
  -> allowed: execute approved action
  -> denied: write denial evidence
  -> halted: write halt evidence
  -> write checkpoint
  -> append manifest checkpoint record
  -> write ledger
```

## Testing

v0.2-beta must prove:

- valid and invalid `ActionIntent` payloads
- `PermissionMatrix` allows `write_text` in `111111`
- `PermissionMatrix` denies `bash_execution` in `111111`
- `PathGuard.write_text()` writes an allowed relative file and returns matching SHA256
- path traversal is halted with `reason: sovereignty_breach`
- root config write is halted with `reason: sovereignty_breach`
- denied tool action writes ledger evidence without physical execution
- manifest preserves earlier checkpoint records when a later checkpoint is written
- v0.1 timeout behavior still returns `status: halted`, `partial: true`, and `reason: http_timeout`

## Acceptance Criteria

The milestone is complete when:

- `python3 -m onecode.cli run "smoke" --workspace .` still works.
- a legal CLI write creates the requested file under the workspace.
- an illegal CLI write such as `../outside.txt` writes no target file and returns `status: halted`, `reason: sovereignty_breach`.
- a denied `bash_execution` intent returns `decision: denied`, `reason: permission_denied`.
- manifest history contains multiple checkpoint records after multiple writes.
- the full test suite passes.

## Non-Goals

v0.2-beta does not implement:

- shell command execution
- real pytest execution
- model or API calls
- TUI rendering
- complete 64-state policy ontology
- general filesystem abstraction
- deletion, append, read, list, chmod, move, or copy APIs
