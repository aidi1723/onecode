# OneCode v0.5 Iching Complete Rule Kernel Design

Date: 2026-05-28
Project root: `<private-development-workspace>`
Status: Approved for implementation

## Goal

Complete the mathematical rule surface of the OneCode Iching control kernel without turning it into a folklore knowledge base. v0.5 must make the core projections explicit and testable:

- yin/yang bit accounting
- four-symbol projection
- trigram to element mapping
- five-element generation and control relations
- table-driven transition decisions
- complete 64-state hexagram records
- deterministic line-flip mutation

The result should keep Runner and resumption code as consumers of `IchingKernel`, not owners of rule logic.

## Scope

v0.5 extends `src/onecode/kernel/hexagram.py` only as the rule authority. Integrations may be adjusted where they already consume `IchingKernel`.

The implementation proceeds in four small batches:

1. Four-symbol projection.
2. Yin/yang balance profile.
3. Five-element matrix and transition refactor.
4. 64 hexagram records and line-flip mutation.

Each batch must be TDD-driven and end with:

```bash
PYTHONPATH=src python3 -m compileall src tests
PYTHONPATH=src python3 -m unittest discover -s tests -v
```

## Non-Goals

v0.5 does not add:

- traditional Chinese hexagram names
- palace assignment
- changing-line text
- oracle text
- external datasets or network lookup
- UI/TUI rendering
- model/API calls
- changes to source hygiene policy

Traditional names and palace systems require separate source verification before they can be treated as kernel data.

## Bit Order

The canonical order remains lowest bit first:

```text
bit 0: first line
bit 1: second line
bit 2: third line
bit 3: fourth line
bit 4: fifth line
bit 5: sixth line
```

`compute_status(outer, inner)` remains:

```python
((outer & 0b111) << 3) | (inner & 0b111)
```

## Four Symbols

The kernel adds a two-bit projection over the six-line state:

```text
00 -> tai_yin
01 -> shao_yang
10 -> shao_yin
11 -> tai_yang
```

`four_symbols(status_code)` returns three records for bit pairs:

- lower pair: bits 0-1
- middle pair: bits 2-3
- upper pair: bits 4-5

Each record includes `pair_index`, `bits`, and `symbol`.

## Yin/Yang Balance

`yin_yang_profile(status_code)` returns:

- `yang_count`
- `yin_count`
- `balance`

Balance categories:

- `pure_yang`: 6 yang
- `yang_excess`: 5 yang
- `balanced`: 3 or 4 yang
- `yin_excess`: 1 or 2 yang
- `pure_yin`: 0 yang

The existing high-yang transition rule must consume this profile rather than calling `bit_count()` directly.

## Five Elements

Trigram element mapping:

```text
QIAN, DUI -> metal
ZHEN, XUN -> wood
KAN       -> water
LI        -> fire
KUN, GEN  -> earth
```

Generation cycle:

```text
wood -> fire -> earth -> metal -> water -> wood
```

Control cycle:

```text
wood -> earth -> water -> fire -> metal -> wood
```

`element_relation(source, target)` returns:

- `generates`
- `controls`
- `same`
- `neutral`

where source and target are element names.

## Transition Matrix

`transition(status_code)` becomes table-driven in behavior even if implemented with small local tables. It must preserve existing externally observed results:

- outer `LI` over any inner state halts and collapses inner to `KUN`.
- outer `KAN` over any inner state checkpoints and preserves the status code as a resume seed.
- `pure_yang` or `yang_excess` triggers cooldown by changing outer trigram to `GEN` while preserving inner trigram.
- all other states continue unchanged.

The transition result remains:

```python
IchingTransition(status_code: int, action: str, reason: str | None)
```

## Hexagram Records

The kernel adds complete generated metadata for all 64 states:

```python
hexagram_record(status_code)
```

The record includes:

- `status_code`
- `binary`
- `outer_trigram`
- `inner_trigram`
- `outer_element`
- `inner_element`
- `element_relation`
- `yin_yang`
- `four_symbols`

The table must cover every integer from 0 through 63.

## Line Mutation

The kernel adds:

```python
flip_line(status_code, line_index)
```

Rules:

- `line_index` is 0 through 5.
- the function flips exactly one bit.
- out-of-range line indexes raise `ValueError`.
- status code inputs are masked to six bits before mutation.

## Testing

The test suite must prove:

- four-symbol projection for representative states
- yin/yang balance categories
- every trigram maps to an element
- generation and control cycles return correct relations
- transition preserves existing v0.5 integration behavior
- all 64 hexagram records exist and are internally consistent
- `flip_line()` changes only the requested line and rejects invalid indexes

Existing sovereignty, resumption, timeout, checkpoint, and CLI tests must remain green.
