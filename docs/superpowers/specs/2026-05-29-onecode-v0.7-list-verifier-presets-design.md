# OneCode v0.7 List Verifier Presets Design

## Goal

Expose the built-in verifier policy presets through a read-only CLI command so local developers can discover valid `init-verifier-policy --preset` values without reading source code.

This is a local-development feature. It does not publish code, push branches, create PRs, write files, or execute verifier commands.

## Scope

In scope:

- A new `list-verifier-presets` CLI subcommand.
- JSON output with preset id, command, cwd, and timeout.
- README documentation for the discovery workflow.

Out of scope:

- Running preset commands.
- Project autodetection.
- Remote or user-defined preset registries.
- TUI integration.
- Changing existing verifier policy generation behavior.

## CLI Surface

```bash
PYTHONPATH=src python3 -m onecode list-verifier-presets
```

Output:

```json
{
  "presets": [
    {
      "id": "python-compileall",
      "command": ["...python", "-m", "compileall", "src", "tests"],
      "cwd": ".",
      "timeout_ms": 30000
    },
    {
      "id": "python-unittest",
      "command": ["...python", "-m", "unittest", "discover", "-s", "tests"],
      "cwd": ".",
      "timeout_ms": 30000
    }
  ]
}
```

The list is sorted by preset id for stable output.

## Testing

Required tests:

- CLI prints stable JSON with both built-in presets.
- The output does not write `.onecode/verifier-policy.json`.
- README documents `list-verifier-presets`.

## Acceptance Criteria

- A developer can discover preset ids from the CLI.
- The command is read-only and does not execute verifier commands.
- Existing `init-verifier-policy` and `run-plan` behavior remains unchanged.
