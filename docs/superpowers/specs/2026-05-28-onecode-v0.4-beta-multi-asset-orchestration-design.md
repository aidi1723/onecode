# OneCode v0.4-beta Multi-Asset Orchestration Design

Date: 2026-05-28
Project root: `<private-development-workspace>`
Status: Approved for implementation planning

## Goal

Extend the runner from single-asset execution to deterministic multi-asset orchestration. A single `onecode run` invocation can request multiple `write_text` assets, and each asset must independently pass the existing sovereignty pipeline:

```text
ActionIntent -> LogosGate.preflight -> Resume Skip Rule -> PathGuard.write_text -> checkpoint
```

This makes v0.3 stateful resumption useful at task scale: already-ready assets are skipped, missing assets are written, and the ledger records the complete turn-by-turn evidence chain.

## Scope

v0.4-beta implements repeated CLI write arguments:

```bash
python3 -m onecode.cli run "continue task" \
  --workspace . \
  --run-id retry-003 \
  --resume-from failed-002 \
  --write-text src/mesh.py="..." \
  --write-text tests/test_mesh.py="..."
```

The old single-asset interface remains supported:

```bash
--write-path src/a.py --write-content "..."
```

If a command mixes `--write-text` with `--write-path` or `--write-content`, the CLI rejects the invocation as an argument conflict.

## Non-Goals

v0.4-beta does not implement:

- JSON plan files
- parallel asset execution
- continue-on-error behavior
- generalized dependency graphs
- model/API calls
- retry backoff
- source-file import such as `path@source.txt`
- automatic latest-run discovery

## CLI Contract

`onecode run` gains a repeatable argument:

```text
--write-text PATH=CONTENT
```

Parsing rules:

- The first `=` separates the path from content.
- Empty path is invalid.
- Empty content is allowed.
- Multiple `--write-text` arguments preserve input order.
- Duplicate paths are allowed in v0.4 and execute in order; later writes can overwrite earlier completed writes unless a resumed ready asset causes a skip.
- `--write-text` cannot be combined with `--write-path` or `--write-content`.

The existing `--write-path` and `--write-content` path remains the compatibility path for one asset.

## Runner Contract

Introduce a multi-asset execution path while keeping the old single-asset behavior stable.

The runner should normalize input into an ordered list of `ActionIntent.write_text(...)` values:

- no write arguments: one `noop` intent, preserving current smoke behavior
- old `--write-path` / `--write-content`: one `write_text` intent
- repeated `--write-text`: one `write_text` intent per argument

Each intent is processed through the same control path:

1. `LogosGate.preflight(context, intent)`
2. If denied, append a denied checkpoint and stop.
3. If halted, append a halted checkpoint and stop.
4. If allowed and the target path is present in `context.resume_state.ready_assets`, append a skipped checkpoint and do not call `PathGuard.write_text()`.
5. If allowed and not resumed, call `PathGuard.write_text()` and append a completed checkpoint.

## Fail-Fast Semantics

Multi-asset execution is fail-fast.

If an asset returns `status: denied` or `status: halted`, the runner stops immediately. Assets already processed remain in the manifest and final ledger. Assets after the failure point are not preflighted and do not touch disk.

The final top-level result mirrors the terminal asset status:

- all assets completed or skipped: `status: completed`, `partial: false`, `reason: null`
- denied asset encountered: `status: denied`, `partial: false`, `reason: permission_denied` or equivalent
- halted asset encountered: `status: halted`, `partial: true`, `reason: sovereignty_breach`, `http_timeout`, or equivalent

## Result And Ledger Shape

The final result gains an `assets` list. Each asset entry records:

```json
{
  "index": 1,
  "status": "completed",
  "partial": false,
  "reason": null,
  "decision": "allowed",
  "intent_type": "write_text",
  "payload": {
    "path": "src/a.py",
    "sha256": "..."
  },
  "resumed": false
}
```

Skipped assets use:

```json
{
  "status": "skipped",
  "reason": "resumed_asset_ready",
  "resumed": true,
  "payload": {
    "path": "src/a.py",
    "sha256": "...",
    "source_run_id": "old-run",
    "source_turn_index": 1
  }
}
```

The final result also includes summary counts:

- `requested_count`
- `completed_count`
- `skipped_count`
- `failed_count`

For compatibility, a single-asset run may continue to expose top-level `payload` and `sha256` fields. Multi-asset callers should use `assets`.

## Checkpoint And Manifest Behavior

Each processed asset creates exactly one checkpoint.

Manifest behavior remains append-only:

- completed writes append `status: completed`
- resumed ready assets append `status: skipped`
- denied or halted assets append their failure status
- unprocessed assets after fail-fast do not create checkpoints

Existing v0.3 metadata remains in force:

- `resumed_from`
- `ready_assets`
- checkpoint SHA256
- source run metadata for skipped assets

## Sovereignty Guarantees

v0.4 must preserve v0.2 and v0.3 guarantees:

- Every physical write still goes through `LogosGate.preflight()`.
- Every physical write still goes through `PathGuard.write_text()`.
- Resume skips happen only after `preflight` returns allowed.
- Path traversal halts before disk IO.
- Bash execution remains denied.
- Timeout behavior remains `status: halted`, `partial: true`, `reason: http_timeout`.
- Invalid resume records are ignored, not fatal.

## Testing

The implementation plan must prove:

- CLI accepts repeated `--write-text` arguments.
- CLI rejects mixed `--write-text` and `--write-path` / `--write-content`.
- Multiple assets write in order and produce multiple checkpoints.
- Resume skips a ready asset and writes a missing asset in the same run.
- Path traversal on asset N halts the run and asset N+1 is not written.
- Existing single-asset tests remain green.
- Existing timeout, PathGuard, PermissionMatrix, and source hygiene tests remain green.

## Acceptance Criteria

The milestone is complete when:

- A single CLI run can write at least two assets with repeated `--write-text`.
- A resumed multi-asset run skips an already-ready asset and writes a missing asset.
- A halted middle asset stops later writes.
- The final ledger includes `assets` and accurate summary counts.
- `PYTHONPATH=src python3 -m compileall src tests` passes.
- `PYTHONPATH=src python3 -m unittest discover -s tests -v` passes.
