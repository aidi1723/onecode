# OneCode v0.7 Local Demo Script Design

## Goal

Add a local demo script that runs the v0.7 reliability workflow end to end in a temporary workspace.

This is a local-development helper. It does not publish code, push branches, create PRs, or require a networked model provider.

## Scope

In scope:

- `scripts/demo_v07.sh`
- Temporary workspace creation by default.
- Optional `WORKSPACE=/path` override for repeatable local inspection.
- Demonstrate:
  - `list-verifier-presets`
  - `init-verifier-policy`
  - `run-plan --verifier python-unittest`
  - `inspect`
  - `list-runs`
- README documentation.

Out of scope:

- Model-backed repair demo.
- Publishing or packaging flows.
- Long-running UI/TUI automation.
- Mutating repository source files.

## Behavior

Default run:

```bash
bash scripts/demo_v07.sh
```

The script:

1. Selects `PYTHON` from `$PYTHON`, `.venv/bin/python`, or `python3`.
2. Creates a temporary workspace unless `WORKSPACE` is set.
3. Writes a small task plan into the workspace.
4. Lists verifier presets.
5. Initializes a verifier policy.
6. Runs the plan with `--verifier python-unittest`.
7. Inspects the run.
8. Lists runs.
9. Prints the workspace path.

The script must fail fast with `set -euo pipefail`.

## Testing

Required tests:

- script exists and is executable.
- script includes the v0.7 CLI commands.
- script runs successfully with current interpreter override.
- README documents `bash scripts/demo_v07.sh`.

## Acceptance Criteria

- A local developer can run one command and see the verified `run-plan` workflow succeed.
- The script leaves evidence in a workspace for inspection.
- The full project verification still passes.
