# OneCode v0.7 Verifier Policy Init Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a local CLI helper that writes safe verifier policy JSON for common `run-plan --verifier` workflows.

**Architecture:** Keep verifier execution unchanged. Add pure policy generation helpers beside the existing verifier module, expose them through a small `init-verifier-policy` CLI subcommand, and document the workflow in README.

**Tech Stack:** Python stdlib `argparse`, `json`, `sys`, existing `load_verifier_policy()`, `unittest`.
