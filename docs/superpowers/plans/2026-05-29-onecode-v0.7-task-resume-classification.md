# OneCode v0.7 Task Resume Classification Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add task-level resume decisions for `run-plan --resume-from` and expose them in ledger and inspect output.

**Architecture:** Add `task_resume.py` as a classifier over existing run evidence. It reads source manifest/ledger, planned assets, verifier specs, and physical hashes, then returns decisions and aggregate Iching transition fields. `run-plan` records those decisions before or after the existing asset resume flow without changing `run_task` skip semantics.

**Tech Stack:** Python 3.11+, stdlib `dataclasses`, `json`, `hashlib`, `pathlib`, existing `unittest` suite.

---

## File Structure

- Create `src/onecode/kernel/task_resume.py`: dataclasses and pure classification helpers.
- Modify `src/onecode/cli.py`: call classifier for `run-plan --resume-from`, stop before writes on halt, append task resume fields to final ledger, expose inspect fields.
- Add `tests/test_task_resume.py`: unit tests for classifier behavior.
- Modify `tests/test_run_plan_cli.py`: integration tests for ready/apply/verify/halt behavior.
- Modify `tests/test_inspect_cli.py`: inspect output includes task resume fields.
