# OneCode Update - Rule-Evidence Absorption Layer / OneCode 更新 - 规则证据吸收层

Date: 2026-06-05

## GitHub Release Copy / GitHub 发布文案

OneCode now includes a rule-evidence absorption layer that turns project
instructions, optional runtime configuration, and recovery advice into bounded
metadata and deterministic control summaries. These inputs are treated as
evidence, not authority: they are folded into the existing state, transition,
dispatch, doctor, Web status, and shell projection surfaces before they can
guide local agent workflows.

OneCode 现在加入规则证据吸收层，可将项目指令、可选运行配置和恢复建议转换为受限元数据与确定性控制摘要。这些输入被视为证据，而不是执行权威：它们必须先折叠进既有状态、转移、调度、doctor、Web 状态和壳层投影面，才能用于引导本地 Agent 工作流。

This update keeps the public contract metadata-first. Doctor and Web project
status expose summaries and hashes by default, not raw project rule content.
Shell and UI adapters should consume `shell_projection.control_state` instead
of inferring control state from raw run evidence.

本次更新保持公开契约以元数据为先。Doctor 与 Web 项目状态默认暴露摘要和哈希，而不是原始项目规则内容。Shell 与 UI 适配器应消费 `shell_projection.control_state`，而不是从原始运行证据中自行推断控制状态。

## What Changed / 更新内容

- Added project instruction and rule-file discovery as bounded project context
  metadata.
  新增项目指令与规则文件发现，并以受限项目上下文元数据形式输出。
- Added runtime configuration inspection with approved effective values and
  redacted diagnostics.
  新增运行配置检查，输出经批准的有效值和脱敏诊断。
- Added advisory recovery policy status for common failure scenarios.
  新增常见失败场景的建议式恢复策略状态。
- Added doctor checks for project context, runtime config, and recovery policy.
  Doctor 新增项目上下文、运行配置和恢复策略检查。
- Added Web project status summaries for project context and runtime config.
  Web 项目状态新增项目上下文和运行配置摘要。
- Added shell projection `control_state` for stable shell/UI consumption.
  新增 shell projection `control_state`，便于壳层和 UI 稳定消费。
- Updated public release documentation and open-source readiness evidence.
  更新公开发布文档和开源准备证据。

## Operator Notes / 操作说明

- External tool rules and repositories remain learning references and evidence
  sources, not runtime authority.
  外部工具规则和仓库仍是学习参考与证据来源，不是运行时权威。
- Raw project rule content is not exposed by default on public status surfaces.
  公开状态面默认不暴露原始项目规则内容。
- Production deployment still requires operator-owned gateway controls,
  authentication, TLS, request-size limits, rate limiting, and secret
  management.
  生产部署仍需要使用方掌控的网关、鉴权、TLS、请求大小限制、限流和密钥管理。
