# 八卦动态流转规则 V2

日期：2026-05-26
定位：Build Mode 的运行时状态转移法典
状态：设计法典；与 [Build Mode 内核运行规则](build-mode-kernel-rules.md) 配套

## 1. 核心结论

八卦不是静态标签，而是运行时状态机。每一次工具意图、退出码、路径解析、流式缓冲区变化，都会触发确定性的状态转移。

最高规则：

```text
状态由证据触发，不由模型口头声明触发。
```

V2 补充规则：

```text
正常变卦沿三维立方体棱边行走；一次只改变工具、上下文、边界三个轴中的一个轴。
```

如果目标状态与当前状态的哈明距离大于 1，运行时必须生成 `TransitionPlanEvidence.edge_path`
作为中间路由证据。只有 `ViolationEvidence` 或明确的行为指纹风险可以触发紧急熔断式跳转。
重复失败会进入动态衰减门：当连续失败摘要高度相似时，失败阈值可以从默认 3 次缩短到 1 次。
跨节点证据必须封装为带时间戳、摘要和签名的 evidence envelope，避免多节点状态撕裂。

证据来源只接受：

- `intent_resolver` 的结构化意图判定。
- `tool_call` 的工具名和参数。
- `scoped_writer` 的路径解析结果。
- `sandbox_runner` 的 `exit_code`、stdout/stderr 摘要和 timeout。
- `path_preflight` 的阻断结果。
- `soft_feedback` 的结构化反馈。
- `archive_guard` 的 manifest/hash 结果。

## 2. 两仪互生规则

系统常态是阴仪：低权限、少工具、少上下文、低成本。只有明确检测到物理改写或执行意图，才允许升入阳仪。

| 规则 | 条件 | 转移 | 结果 |
| --- | --- | --- | --- |
| 由静入动 | 请求包含 `mkdir`、写文件、补丁、安装依赖、测试、构建等意图 | 阴 `0` -> 阳 `1` | 激活 Build Mode 生产链 |
| 动极生静 | 测试通过、归档完成、或发生致命熔断 | 阳 `1` -> 阴 `0` | 进入归档、摘要、只读或阻断状态 |
| 阴中求阳 | 纯咨询过程中出现明确修改请求 | 巽/问 -> 乾/造 | 从零工具快轨升入 scoped 写入 |
| 阳中守阴 | 写入、测试或修复过程中出现越权 | 任意阳态 -> 艮/停 | 收回全部权限，进入熔断 |

## 3. 四象跃迁规则

四象决定动作发生在哪个物理空间，不直接等同于八卦，但约束八卦可用能力。

| 跃迁 | 触发 | 结果 |
| --- | --- | --- |
| 少阳 `01` -> 太阳 `11` | 纯推理后出现 `write_file`、`make_dir`、`patch` 意图 | 进入 `乾 111`，开放 scoped 写入 |
| 太阳 `11` -> 少阴 `10` | 写入完成后出现 `run_pytest`、构建、安装依赖意图 | 进入 `震 001`，开放隔离执行 |
| 任意空间 -> 太阴 `00` | 路径逃逸、危险命令、密钥读取、宿主机破坏 | 进入 `艮 100`，物理熔断 |
| 少阴 `10` -> 少阳 `01` | 测试失败或阻断结果需要模型理解 | 进入 `兑 110` 后转 `离 101` |
| 少阴 `10` -> 太阴 `00` | 测试通过并准备收工 | 进入 `坤 000`，归档收束 |

## 4. 八卦状态转移矩阵

| 当前状态 | 触发证据 | 下一状态 | 说明 |
| --- | --- | --- | --- |
| `巽 011` 问 | 纯文本回答完成 | 结束或 `总` | 无工具、无写入 |
| `巽 011` 问 | 出现明确写入意图 | `乾 111` 造 | 从轻轨升入 Build Mode |
| `离 101` 查 | repo card 生成且有可修复目标 | `乾 111` 造 | 注入上下文后给 scoped 写权 |
| `离 101` 查 | 只读审查完成，无需改动 | `总` | 输出摘要 |
| `乾 111` 造 | 写入完成，`changed_files` 非空 | `震 001` 测 | 收回写权，转验证 |
| `乾 111` 造 | 路径逃逸或敏感路径 | `艮 100` 停 | 写入前阻断 |
| `乾 111` 造 | 无实际文件变化 | `兑 110` 纠 | 要求补充可验证动作 |
| `震 001` 测 | `exit_code == 0` | `坤 000` 归 | 测试通过，进入归档 |
| `震 001` 测 | `exit_code != 0` 且未超限 | `兑 110` 纠 | 失败压缩反馈 |
| `震 001` 测 | timeout、OOM、连续失败超限 | `艮 100` 停 | 停止重试 |
| `艮 100` 停 | 阻断事件已结构化 | `兑 110` 纠 | 将硬拒绝转成软反馈 |
| `艮 100` 停 | 需要人类授权 | V1 `问` | 等待人工确认 |
| `兑 110` 纠 | 反馈 JSON 生成成功 | `离 101` 查 | 注入错误上下文 |
| `兑 110` 纠 | 反馈生成失败 | `艮 100` 停 | 防止错误继续扩散 |
| `坎 010` 隔 | 风险解除且有明确修复路径 | `兑 110` 纠 | 转为可处理反馈 |
| `坎 010` 隔 | 风险无法识别或持续震荡 | `艮 100` 停 | 熔断 |
| `坤 000` 归 | manifest/hash 成功 | V1 `总` | 进入最终交付摘要 |
| `坤 000` 归 | manifest/hash 失败 | `艮 100` 停 | 归档失败不能声称完成 |

## 5. 各卦硬规则

### 5.1 乾 `111`：造

触发：

- 初始任务被判定为项目构建、功能开发、修复。
- `离` 已经注入可修复上下文。

允许：

- `make_dir`
- `write_file`
- `patch`
- 读取当前工作区中与目标有关的最小文件片段

禁止：

- 工作区外路径。
- shell 绕过写入。
- 读取密钥、用户配置和系统目录。
- 在乾态直接运行测试或安装依赖。

转移：

```text
changed_files 非空 -> 震 001
路径越权 -> 艮 100
无文件变化 -> 兑 110
```

### 5.2 震 `001`：测

触发：

- `乾` 完成写入。
- 用户明确要求验证或测试。

允许：

- 在隔离环境运行测试、构建、必要依赖安装。
- 捕获 stdout/stderr 摘要。
- 记录退出码和耗时。

禁止：

- 在宿主机裸跑高风险命令。
- 在测试态修改业务源码。
- 把完整日志原样塞回模型。

转移：

```text
exit_code == 0 -> 坤 000
exit_code != 0 -> 兑 110
timeout/OOM/失败超限 -> 艮 100
```

### 5.3 兑 `110`：纠

触发：

- `震` 测试失败。
- `艮` 阻断越权。
- `乾` 无有效写入。

允许：

- 将失败压缩为结构化 JSON。
- 保持客户端连接语义连续。
- 给出下一状态建议。

禁止：

- 伪造成功。
- 吞掉证据。
- 倾倒海量日志。

转移：

```text
feedback_json 成功 -> 离 101
feedback_json 失败 -> 艮 100
```

### 5.4 离 `101`：查

触发：

- `兑` 已生成失败反馈。
- 初始任务是只读审查或上下文定位。

允许：

- 生成 repo card。
- 提取目标文件、行号、失败断言、最小上下文。
- 注入 System 域。

禁止：

- 修改文件。
- 安装依赖。
- 执行测试。

转移：

```text
有可修复目标 -> 乾 111
只读任务完成 -> 总
发现高风险 -> 坎 010 或 艮 100
```

### 5.5 艮 `100`：停

触发：

- 路径逃逸。
- 危险命令。
- 密钥或系统配置访问。
- timeout/OOM/连续失败超限。
- 归档失败。

允许：

- 阻断动作。
- 冻结现场。
- 输出恢复条件。

禁止：

- 继续执行危险工具。
- 为了完成任务绕过策略。

转移：

```text
可软化的阻断 -> 兑 110
需要授权 -> V1 问
不可恢复 -> 结束
```

### 5.6 坤 `000`：归

触发：

- `震` 返回 `exit_code == 0`。

允许：

- 清理瞬时容器和临时文件。
- 生成 manifest。
- 计算 SHA256。
- 在 `lockdown` 模式下只读固化。

禁止：

- 新增业务改动。
- 继续运行测试。
- 未生成 manifest 就声称完成。

转移：

```text
manifest/hash 成功 -> V1 总
manifest/hash 失败 -> 艮 100
```

说明：MVP 默认使用 `audit_only` 归档模式，只生成 manifest/hash，不执行 `chmod 444`。`lockdown` 只用于高安全交付场景。

### 5.7 巽 `011`：问

触发：

- 初始任务是纯咨询、纯分析、纯问答。
- 低置信度但不需要立即写盘。

允许：

- `tools=[]` 的纯文本回答。
- 必要时向用户提出澄清问题。

禁止：

- 写盘。
- 执行命令。
- 伪造读盘结果。

转移：

```text
纯文本完成 -> 结束或 V1 总
出现明确写入意图 -> 乾 111
需要人类确认 -> V1 问
```

### 5.8 坎 `010`：隔

触发：

- 未知风险。
- 高频震荡。
- I/O 意图不明确但可能有破坏性。

允许：

- 影子文件系统。
- RAM buffer。
- 风险缓冲与观察。

禁止：

- 真实写盘。
- 真实外联。
- 宿主机执行。

转移：

```text
风险可解释 -> 兑 110
风险不可解释 -> 艮 100
风险解除且只读 -> 离 101
```

## 6. 事件优先级

当多个事件同时出现，按以下优先级处理：

1. 越权或危险命令：立即 `艮 100`。
2. timeout/OOM/连续失败超限：立即 `艮 100`。
3. 测试退出码：`0 -> 坤`，非零 -> `兑`。
4. 写入完成：`乾 -> 震`。
5. 只读上下文完成：`离 -> 总` 或 `巽 -> 结束`。
6. 低置信度：进入 V1 `问`。

高优先级事件会覆盖低优先级事件。例如测试失败日志中同时包含危险命令企图时，先进入 `艮`，再由 `兑` 反馈。

## 7. 计数器与预算

Build Mode 必须维护以下计数器：

| 计数器 | 默认阈值 | 超限结果 |
| --- | --- | --- |
| `fix_attempts` | 3 | `艮 100` |
| `sandbox_timeouts` | 1 | `艮 100` |
| `blocked_actions` | 1 | `艮 100` 后 `兑 110` |
| `context_card_tokens` | 目标小于 800 tokens | 超限重新压缩 |
| `stdout_chars` | 按任务配置，默认截断 | 进入 `兑 110` 压缩 |

## 8. 完成断言

Build Mode 只有满足以下全部条件，才允许输出完成：

```text
changed_files 非空
路径全部在 workspace scope 内
sandbox_runner 执行过
exit_code == 0
archive_guard 生成 manifest
manifest 包含每个交付文件的 SHA256
V1 总 输出最终摘要和剩余风险
```

不满足任何一项，都只能输出“未完成”或“等待修复/等待授权”，不能输出成功。

## 9. 与实现的关系

本文档定义运行时状态转移规则，不代表当前代码已经全部实现。

实现顺序建议：

1. 先实现状态枚举和转移表。
2. 再实现 `intent_resolver` 与 `permission_matrix`。
3. 再接 `scoped_writer`、`sandbox_runner`、`soft_feedback`。
4. 最后接 `archive_guard` 和 V1 `总` 的最终交付摘要。

下一轮 A/B 测试必须在这些规则至少完成 MVP 闭环后再进行。
