# 总 Opcode Workflow

## Purpose

压缩上下文、生成交接摘要、保留事实、风险、证据和下一步执行字。

## Prompt Engineering Sources

- Context compaction pattern: 保留可执行状态，丢弃闲聊和重复推理。
- Handoff summary pattern: 让下一个 Agent 能直接接续工作。
- Evidence-preserving summary pattern: 摘要必须保留验证命令、文件、风险和未完成事项。

## Root Skill Mount

- 挂载名：`handoff_compaction_mount`
- 社区参考源：Claude Code context management、Agent handoff summary pattern。
- 挂载动作：加载已完成事实、未解决风险、验证证据和推荐下一执行字。
- 工具倾向：summarize state、只读文件读取、代码检索。
- 硬门：禁止把计划写成已完成事实；禁止丢弃阻塞项和失败证据。

## Efficiency Controls

- 把长上下文压缩为事实、变更、证据、风险和下一步。
- 只保留会影响后续执行的内容。
- 给出推荐下一执行字，减少下一轮重新分类成本。

## Precision Controls

- 明确区分已完成、未完成、失败、阻塞和待确认。
- 验证结果必须引用实际命令或证据摘要。
- 不把计划写成已完成事实。

## Stability Controls

- 默认不修改业务文件，只生成可交接摘要。
- 上下文压缩后必须保留原始需求、关键决策和证据索引。
- 遇到不确定状态时转 `问` 或 `查`，不强行收口。

## Six Phases

1. 收集已完成事实。
2. 收集未完成事项、风险和失败证据。
3. 丢弃冗余对话和临时推断。
4. 生成结构化摘要。
5. 推荐下一执行字。
6. 输出可交接状态。
