# 记 Opcode Workflow

## Purpose

记录项目记忆、ADR、规则和长期可复用上下文，不记录未验证猜测。

## Prompt Engineering Sources

- CLAUDE.md / AGENTS.md project memory pattern: 把稳定规则放入项目上下文。
- ADR pattern: 记录决策、理由、影响和替代方案。
- Docs-as-source-of-truth pattern: 只把已确认事实写入长期文档。

## Root Skill Mount

- 挂载名：`memory_bank_mount`
- 社区参考源：Claude Code memory、Architecture Decision Record。
- 挂载动作：加载已验证事实、架构变更、技术债和环境快照。
- 工具倾向：append knowledge base、write markdown doc、sha256 file。
- 硬门：禁止把未验证推断写成规则；只能写指定知识库、ADR 或 Markdown 路径。

## Efficiency Controls

- 记录短而稳定的事实，不把完整对话复制进记忆。
- 优先更新已有文档或索引，避免重复信息散落。
- 用摘要、路径和哈希替代长上下文常驻。

## Precision Controls

- 区分事实、决策、偏好、待办和假设。
- 每条记忆必须有来源或上下文引用。
- 未验证推断只能进入待确认区，不能写成规则。

## Stability Controls

- 写权限限制在指定知识库、文档或 ADR 路径。
- 修改项必须可审计，包含写入位置和文件哈希。
- 与现有规则冲突时转 `问`，不得擅自覆盖。

## Six Phases

1. 确认要记录的事实、规则或决策。
2. 检查是否已验证且适合长期保存。
3. 选择写入位置。
4. 写入简短结构化内容。
5. 捕获文件哈希。
6. 输出记录位置和后续引用方式。
