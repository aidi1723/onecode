# OneCode v0.7 Run-Plan Verifier Repair Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add opt-in patch-only repair for `run-plan` controlled verifier failures.

**Architecture:** Add focused repair helpers near the existing CLI verifier path and reuse existing model-provider and patch execution primitives from `model_loop.py`. The repair loop runs only after controlled verifier failure, accepts only patch-only model plans, reruns selected verifiers after each patch attempt, and records repair evidence in the ledger and inspect summary.

**Tech Stack:** Python 3.11+, stdlib `argparse`, existing `ModelPlan` provider protocol, existing `execute_model_plan`, `unittest`.

---

## File Structure

- Modify `src/onecode/cli.py`: add repair flags, repair prompt/evidence helpers, repair loop, inspect fields.
- Modify `tests/test_run_plan_cli.py`: integration tests for successful repair, rejected repair plan, exhausted attempts, and no-op repair disabled.
- Modify `tests/test_inspect_cli.py`: inspect surfaces repair evidence.
- No new model provider schema is needed; tests will use mocked providers.
