# OneCode v0.7 Default Verifier Policy Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let `run-plan --verifier ID` use the workspace `.onecode/verifier-policy.json` when `--verifier-policy` is omitted.

**Architecture:** Keep verifier loading in the CLI preflight path before any asset writes. Add a small resolver helper that chooses explicit policy first and otherwise derives the default path from the workspace.

**Tech Stack:** Python stdlib `pathlib`, existing verifier loader, `unittest`.
