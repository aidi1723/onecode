# OneCode v0.7 Default Verifier Policy Design

## Goal

Reduce local `run-plan --verifier` friction by using the workspace default verifier policy file when a verifier id is selected and `--verifier-policy` is omitted.

This is a local-development convenience. It does not publish code, push branches, create PRs, auto-select verifiers, or execute any verifier unless the user explicitly passes `--verifier`.

## Scope

In scope:

- `run-plan` only.
- Default policy path: `.onecode/verifier-policy.json` inside the selected workspace.
- Preserve explicit `--verifier-policy PATH` precedence.
- Parser error before writes when a selected verifier cannot be resolved from either explicit or default policy.
- README update showing the shorter workflow after `init-verifier-policy`.

Out of scope:

- Auto-running default verifiers with no `--verifier`.
- Searching parent directories.
- Global or user-level verifier policy.
- TUI integration.
- Changing verifier execution semantics.

## Behavior

When `run-plan` receives one or more `--verifier ID` values:

1. If `--verifier-policy PATH` is present, load that path exactly as today.
2. If `--verifier-policy` is omitted, load `<workspace>/.onecode/verifier-policy.json`.
3. If the selected policy file is missing or invalid, fail through the existing parser error path before asset writes.
4. Validate selected verifier ids and verifier cwd before writes, as today.

When `--verifier` is omitted, no verifier policy is loaded and no default verifier runs.

## Testing

Required tests:

- `run-plan --verifier python-unittest` uses `<workspace>/.onecode/verifier-policy.json` when no explicit policy is passed.
- explicit `--verifier-policy` overrides the workspace default.
- missing default policy errors before asset writes.
- `run-plan` without `--verifier` does not require or load default policy.
- README documents the short workflow.

## Acceptance Criteria

- A local developer can run `init-verifier-policy` once, then use `run-plan --verifier python-unittest` without repeating `--verifier-policy`.
- No verifier runs unless selected with `--verifier`.
- Existing explicit policy behavior remains unchanged.
